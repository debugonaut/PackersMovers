\section{Project Overview}
SwiftMove is a modern, high-performance landing page tailored for a Packers and Movers business. Built as an alternative to traditional WordPress setups, it delivers superior performance, customization, and security. This project is a submission for a Web Developer Intern assignment, demonstrating deep technical capability in both frontend UI/UX and backend API development. It utilizes React + Vite on the frontend and Node.js + Express on the backend, ensuring a production-ready, scalable architecture.

The codebase has been refactored into a clean monorepo structure. All legacy, redundant, static root HTML/CSS assets have been removed, and the deployment is managed via a single Vercel project configuration linking the React client and Express server under a unified domain.

\section{Features}
\begin{itemize}
    \item \textbf{Flat Outline Vector Illustration Aesthetic}: Shifted from the dark/orange template to a premium, retro-flat design using HSL cream background (\texttt{\#FBF5DD}), pale green cards (\texttt{\#E7E1B1}), dark green borders (\texttt{\#0D530E}), and thick offsets.
    \item \textbf{Fully Responsive Mobile-First Design}: Adapts seamlessly to all screen sizes.
    \item \textbf{Sticky Animated Navbar}: Features scroll detection for dynamic cream background rendering.
    \item \textbf{Framer Motion Animations}: Includes staggered animations, retro pop reveals, and hover offsets.
    \item \textbf{6 Interactive Service Cards}: Upgraded from five to six fully integrated services by adding \textbf{Packing \& Unpacking} with customized detail assets.
    \item \textbf{Emoji-to-SVG Transition}: Replaced all standard emojis (like 🎉, 🚚, and ❤️) with clean, high-quality, scalable SVG icons from the \texttt{react-icons} library for premium visual control.
    \item \textbf{Validated Contact Form}: Robust validation using Zod schemas on the frontend and express-validator checks on the backend.
    \item \textbf{Rate Limiting}: Protects against email spam with IP-based caps (5 requests per 15 minutes).
    \item \textbf{Resend API with Lazy Loading}: Engineered an asynchronous environment-variable import resolution that lazily constructs the Resend SDK client to guarantee bulletproof initialization in serverless environments.
    \item \textbf{SEO \& Social Metadata}: Implements Open Graph, Twitter Cards, and JSON-LD LocalBusiness structured data.
    \item \textbf{Accessibility Compliance}: Utilizes ARIA attributes and semantic HTML to ensure usability for all.
    \item \textbf{Optimized Production Build}: Manual chunk splitting for caching and performance benefits.
\end{itemize}

\section{Getting Started — Local Development}
\begin{enumerate}
    \item Clone the repository and navigate into the project root.
    \item Install server dependencies: \texttt{cd server \&\& npm install}
    \item Install client dependencies: \texttt{cd ../client \&\& npm install}
    \item Copy the environment file: \texttt{cp .env.example .env}
    \item Fill in all environment variable values (explained in the next section).
    \item Open two terminal windows:
    \begin{itemize}
        \item In the first, run the backend: \texttt{cd server \&\& npm run dev}
        \item In the second, run the frontend: \texttt{cd client \&\& npm run dev}
    \end{itemize}
    \item Open \texttt{http://localhost:5173} in a browser.
\end{enumerate}

\section{API Reference}
\subsection*{Backend API}

\textbf{\texttt{GET /api/health}}
\begin{itemize}
    \item \textbf{Description}: Health check endpoint. No request body, returns \texttt{\{ status: 'ok', timestamp: '...', environment: '...' \}}, HTTP 200. Used for deployment health checks.
\end{itemize}

\textbf{\texttt{POST /api/contact}}
\begin{itemize}
    \item \textbf{Description}: Request body is JSON with fields: \texttt{name} (string, required), \texttt{phone} (string, required, 10-digit Indian mobile), \texttt{service} (string, required, one of six enum values), \texttt{message} (string, optional, max 500 characters).
    \item \textbf{Responses}:
        \begin{itemize}
            \item Success response HTTP 200: \texttt{\{ success: true, message: '...' \}}
            \item Validation error HTTP 422: \texttt{\{ errors: [...] \}}
            \item Rate limit error HTTP 429: \texttt{\{ error: '...' \}}
            \item Server error HTTP 500: \texttt{\{ error: '...' \}}
        \end{itemize}
\end{itemize}

\section{Monorepo Vercel Deployment}
Rather than configuring separate Vercel projects for the frontend and backend, the entire monorepo is deployed together in a single Vercel project under one unified domain. This is made possible by the newly introduced root-level \texttt{vercel.json} configuration:

\begin{lstlisting}[language=json, caption=Root-level vercel.json]
{
  "version": 2,
  "builds": [
    {
      "src": "client/package.json",
      "use": "@vercel/static-build",
      "config": { "distDir": "dist" }
    },
    {
      "src": "server/index.js",
      "use": "@vercel/node"
    }
  ],
  "routes": [
    {
      "src": "/api/(.*)",
      "dest": "server/index.js"
    },
    {
      "src": "/(.*)",
      "dest": "client/$1"
    }
  ]
}
\end{lstlisting}

This setup automatically triggers the Vite React build inside \texttt{client/}, bundles all static assets, compiles the Express \texttt{server/index.js} entry point as a serverless Node.js function, and routes all \texttt{/api/*} endpoints to the backend while routing all other requests to the React SPA.

\section{Technical Architecture}

\subsection{System Overview}
The SwiftMove project operates as a monorepo containing two distinct workspaces: a Vite React client and an Express Node.js API. These are unified in production by the Vercel CDN router. Frontend form queries bypass cross-origin security concerns since both client and server are hosted on the same domain.

\subsection{Architecture Diagram}
\begin{verbatim}

                                |
                                v

         | (All other routes)                          | (Paths matching /api/*)
         v                                             v

  [Theme: Cream & Green HSL]                   [Lazy-Loaded Resend API]

\subsection{Resend Email System & Lazy Loading}
To avoid initialization race conditions in serverless runtimes where static imports evaluate before environment configurations (such as \texttt{dotenv.config()}) take effect, the Resend client is initialized lazily:

\begin{lstlisting}[language=JavaScript, caption=Lazy Initialization of Resend SDK]
export const sendEnquiry = async (enquiryData) => {
  // Lazily construct the Resend instance inside the handler
  const resend = new Resend(process.env.RESEND_API_KEY);
  
  return await resend.emails.send({
    from: process.env.EMAIL_FROM,
    to: process.env.EMAIL_TO,
    subject: `New Enquiry from ${enquiryData.name}`,
    html: generateEmailTemplate(enquiryData)
  });
};
\end{lstlisting}

\subsection{Premium Email Notification Template}
The HTML template was fully redesigned to be robust against Dark Mode color inversion in visual email clients (like Gmail and Outlook). It employs a solid dark slate background card (\texttt{\#111827}), highly visible contrast colors (\texttt{\#F3F4F6} for headers, \texttt{\#9CA3AF} for labels), and a customized brand badge (\texttt{SM}) utilizing standard CSS parameters to bypass SVG blocking policies.

\subsection{Visual Design Strategy}
The flat outline art style uses high-contrast borders (\texttt{2px solid \#0D530E}) and solid offset drop shadows rather than radial gradients or Gaussian blurs. This creates a clean, illustrative vector aesthetic that is lightweight, highly legible, and performant. Emojis were replaced with SVG vectors from the \texttt{react-icons} library to maintain this clean art-style throughout the interface.
